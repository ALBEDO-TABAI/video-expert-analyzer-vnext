## SVG 动态图形形式结构图 — AGENT PROMPT

### 任务描述

根据你完成的动态图形分析报告，生成一张
**形式元素（视觉材料）/ 运动编排（动态行为）/ 音画效果（感受生产）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是**形式的因果生成链**：
每一行代表动态图形的一个节奏段落，三列分别回答——
- 左列：这一段落**投入了什么视觉材料**？（几何体/字体/色块/粒子/图标……）
- 中列：这些材料被施加了**什么运动行为**？处于全片运动弧线的哪个阶段？
- 右列：这些运动行为与声音轨道共同**生产了什么感受/意义**？

> **三栏锚定声明**：
> - 左列锚定 MODULE 1（视觉元素系统盘点）
> - 中列锚定 MODULE 2（运动行为分析）
> - 右列锚定 MODULE 3（音画关系解剖）+ MODULE 4（时间结构与节奏）

---

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

---

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  形式元素（视觉材料） |  运动编排（动态行为） |  音画效果（感受生产）
[分隔线]
[段落 1 行]  左卡片  →  中卡片  →  右卡片
[段落 2 行]  左卡片  →  中卡片  →  右卡片
...
[段落 N 行]  左卡片  →  中卡片  →  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）

### 卡片规格

每个卡片为双行文字卡片：

```svg
<g class="node c-{COLOR}">
  <rect x="{X}" y="{Y}" width="168" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="{X+84}" y="{Y+19}"
        text-anchor="middle" dominant-baseline="central">
    主标题（≤10字）
  </text>
  <text class="ts" x="{X+84}" y="{Y+39}"
        text-anchor="middle" dominant-baseline="central">
    副标题（≤16字）
  </text>
</g>
```

卡片点击绑定 sendPrompt，内容为"详细分析 [主标题] 段落"：
```svg
<g class="node c-{COLOR}" onclick="sendPrompt('详细分析 [主标题] 段落')">
```

### 颜色分配规则

每一个节奏段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**运动节奏阶段的动态特征**，而非视觉装饰。按以下逻辑选色：

| 运动阶段特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| 静启/生长/元素初现 | c-coral | 从无到有的暖调能量释放 |
| 铺展/系统建立/节奏锚定 | c-teal | 冷静的结构搭建阶段 |
| 加速/密度攀升/运动叠加 | c-pink | 能量升温，复杂度上升 |
| 爆发/密度峰值/视觉高潮 | c-purple | 最高运动密度的集中爆破点 |
| 减速/消解/元素归位 | c-green | 运动能量的有序释放与着陆 |
| 静止/品牌落款/尾帧锁定 | c-gray | 从运动退出到静态收束 |
| 可根据实际运动弧线灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各节奏段落）。

### 连接箭头规格

**水平箭头**（同行，左→中→右，表示因果生成方向）：

```svg
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图与箭头方向论证**：水平箭头在 MV 图中是"对照关系"（画面 ↔ 主题 ↔ 语言），
> 但在动态图形图中是**因果生成链**——视觉材料经由运动编排生成感受效果。
> 这是一条单向的"生产"关系：材料被施加运动→运动与声音共同产生效果。
> 因此箭头方向严格为 **左→中→右（单向）**，不可反向。
> 这与广告图的意义转移方向逻辑一致，但因果内容不同：
> 广告是"文化意义→修辞机制→品牌价值"，
> 动态图形是"形式材料→运动行为→视听感受"。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示时间/节奏进程的推进。
> 左列的垂直箭头 = 视觉元素的引入/叠加/替换序列；
> 中列的垂直箭头 = 运动密度与复杂度的弧线递进；
> 右列的垂直箭头 = 感受强度的累积/转换/消解。

### 内容填充规则

根据你的动态图形分析报告，将作品的时间轴划分为 3–6 个节奏段落
（不是机械等分时间，而是按**运动密度变化点、音画同步类型转换点、
转场断裂点**来切分）。为每个段落填写三张卡片：

**左卡片（形式元素 — 视觉材料）**
- 主标题：该段落的主导视觉元素类型（≤8字）
  例如："几何粒子群" "品牌字体组" "数据图表层" "渐变色块"
- 副标题：空间类型 + 主导色彩/形状特征（≤14字）
  例如："平面空间·蓝灰冷调" "深空间·高饱和暖色"

**中卡片（运动编排 — 动态行为）**
- 主标题：该段落的主导运动行为/运动性格（≤8字）
  例如："弹性生长" "交错滑入" "脉冲爆发" "匀速漂浮" "减速归位"
- 副标题：时间戳范围，格式 "MM:SS – MM:SS"

**右卡片（音画效果 — 感受生产）**
- 主标题：该段落生产的核心感受/功能效果（≤10字）
  例如："节奏锚定·专注感" "紧迫感·信息锤击" "静谧·品牌气质沉淀"
- 副标题：音画同步类型（≤14字）
  例如："逐帧对应·强节拍锁定" "情绪共振·氛围铺底" "静默独白·视觉收束"

> **填充原则**：
> 1. 左列写"这一段投入了什么视觉材料"——锚定 MODULE 1 的元素清单
> 2. 中列写"这些材料被怎样运动"——锚定 MODULE 2 的运动行为与运动性格
> 3. 右列写"运动+声音共同生产了什么效果"——锚定 MODULE 3 的音画关系类型
>    + MODULE 4 的节奏密度判定
> 4. 如果某段落为纯音乐/环境音铺底且视觉元素尚未出现（如开场黑屏+声音），
>    左列卡片的主标题写"（待充能）"，副标题写"元素尚未入场"

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **运动密度断裂点**：视觉事件密度发生陡变的时刻（从稀疏到密集，或从密集到静止）
2. **音画同步类型切换点**：从逐帧对应切到情绪共振，或从节奏锚定切到静默
3. **转场断裂点**：明确的段落转场（硬切/全局色调变化/元素清场后重建）
4. **元素系统替换点**：主导视觉元素发生根本性更换（如从几何图形切到字体动画）
5. **品牌/信息着陆点**：logo 或核心信息首次出现/最终锁定的时刻
6. **循环边界**：如果作品包含 loop 结构，循环的起止点为自然切分点

典型的动态图形段落划分：

| 子类型 | 典型段落数 | 典型结构 |
|----------|-----------|---------|
| 5s Logo Sting | 2–3 段 | 元素聚合 → 品牌成型 → 静态锁定 |
| 15s 社交媒体动画 | 3 段 | 钩子/冲击 → 信息展示 → 品牌收束 |
| 30s 解释型动画 | 4 段 | 问题引入 → 机制展示 → 方案着陆 → CTA |
| 60s 品牌动态图形 | 4–5 段 | 氛围建立 → 元素铺展 → 运动高潮 → 品牌揭示 → 标语收束 |
| 2–3min 数据可视化动画 | 5–6 段 | 数据引入 → 趋势展示 → 对比/高亮 → 洞察揭示 → 结论着陆 → 品牌收束 |
| 抽象/艺术型 Loop | 3–4 段 | 生长 → 密度攀升 → 高潮/爆发 → 消解/循环回归 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：5 段 → 50 + 5×90 + 20 = 520
示例：3 段 → 50 + 3×90 + 20 = 340
示例：6 段 → 50 + 6×90 + 20 = 610

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为双向——因果生成方向严格为 左→中→右
- 禁止在右列"音画效果"卡片中写元素描述——右列只写运动+声音共同生产了什么感受/效果
- 禁止将动态图形的段落按"叙事幕"来切分——应按运动密度变化和音画同步转换来切分
