## SVG 信息动画认知架构图 — AGENT PROMPT

### 任务描述

根据你完成的信息动画分镜分析报告，生成一张
**信息输入（内容层）/ 认知引导（设计层）/ 心智建构（效果层）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是 Mayer（2001）的**双通道认知建构模型**：
每一行代表信息动画的一个认知段落，三列分别回答——
- 左列：这一段传达了**什么信息单元**？（概念/数据/机制/类比）
- 中列：这一段使用了什么**认知引导策略**？处于认知弧线的哪个阶段？
- 右列：这一段在观众脑中**建构了什么心智模型变化**？

> **三栏锚定说明**：
> 左列锚定 MODULE 2（信息架构 — 信息单元清单）
> 中列锚定 MODULE 3（认知引导弧线）+ MODULE 1（动态视觉语法）
> 右列锚定 MODULE 6（认知负荷地形图）+ MODULE 3（知识状态追踪）

---

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

---

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  信息输入（内容层） |  认知引导（设计层） |  心智建构（效果层）
[分隔线]
[段落 1 行]  左卡片  →  中卡片  →  右卡片
[段落 2 行]  左卡片  →  中卡片  →  右卡片
...
[段落 N 行]  左卡片  →  中卡片  →  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）

### 卡片规格

每个卡片为双行文字卡片：

```svg
<g class="node c-{COLOR}">
  <rect x="{X}" y="{Y}" width="168" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="{X+84}" y="{Y+19}"
        text-anchor="middle" dominant-baseline="central">
    主标题（≤10字）
  </text>
  <text class="ts" x="{X+84}" y="{Y+39}"
        text-anchor="middle" dominant-baseline="central">
    副标题（≤16字）
  </text>
</g>
```

卡片点击绑定 sendPrompt，内容为"详细分析 [主标题] 段落"：
```svg
<g class="node c-{COLOR}" onclick="sendPrompt('详细分析 [主标题] 段落')">
```

### 颜色分配规则

每一个认知段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**认知功能阶段**，而非视觉装饰。按以下逻辑选色：

| 认知阶段特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| 钩子/知识缺口制造/问题提出 | c-coral | 高能量开场，唤起认知动机 |
| 概念锚定/前置知识激活/术语引入 | c-teal | 冷静的基础建设阶段，Mayer 预训练原则 |
| 数据呈现/证据展示/信息密集段 | c-purple | 认知负荷高峰区，高密度信息集中点 |
| 机制解释/因果拆解/核心论证 | c-pink | 理解的关键转化段，认知脚手架搭建 |
| 整合/模型建构完成/"啊哈时刻" | c-green | 心智模型闭合 = 理解"变现" |
| 总结回顾/认知闭合/CTA | c-gray | 克制收尾，从理解退出到行动/记忆固化 |
| 可根据实际认知弧线灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各认知段落）。

### 连接箭头规格

**水平箭头**（同行，左→中→右，表示认知建构方向）：

```svg
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图**：水平箭头在 MV 图中是"对照关系"（画面 ↔ 主题 ↔ 语言），
> 但在信息动画图中是**认知建构流向**——
> 信息内容经过设计编码，在观众脑中建构心智模型。
> 这是一个单向因果过程：内容→编码→认知效果。
> 因此箭头方向严格为 左→中→右（单向），不可反向。
>
> **论证**：信息动画与 MV 的本质差异在于——MV 的画面与歌词是"平行互注"的双线关系，
> 而信息动画的内容-设计-效果是"串联因果"的管线关系。
> 同一个信息单元，被不同的设计策略编码，会产生截然不同的认知效果。
> 箭头的单向性表达的正是这种"设计决策决定认知效果"的因果方向。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示时间/认知进程的推进。
> 左列的垂直箭头 = 信息单元的递进/深化（从简单到复杂，从已知到未知）；
> 中列的垂直箭头 = 认知引导策略的阶段演进（从钩子到闭合）；
> 右列的垂直箭头 = 观众心智模型的逐步建构（从 S₀ 到 Sₙ 的知识状态累积）。

### 内容填充规则

根据你的信息动画分析报告，将时间轴划分为 3–6 个认知段落
（不是机械等分时间，而是按**认知功能的转换**来切分——
即观众的认知任务发生质变的时刻）。
为每个段落填写三张卡片：

**左卡片（信息输入 — 内容层）**
- 主标题：该段落传达的核心信息单元名称（≤8字）
  例如："问题现象" "核心概念定义" "关键数据集" "运作机制" "解决方案"
- 副标题：信息单元类型 + 复杂度标记（≤14字）
  例如："概念型·低复杂度" "数据型·6组对比数据" "机制型·三步因果链"

**中卡片（认知引导 — 设计层）**
- 主标题：该段落在认知弧线中的功能角色（≤8字）
  例如："知识缺口制造" "概念锚定" "证据展示" "因果拆解" "模型整合" "认知闭合"
- 副标题：时间戳范围，格式 "MM:SS – MM:SS"

**右卡片（心智建构 — 效果层）**
- 主标题：该段落在观众脑中建构了什么（≤10字）
  例如："认知动机激活" "基础框架建立" "数据模式识别" "因果模型形成" "完整心智模型"
- 副标题：认知负荷评估 + Mayer 原则合规性简注（≤14字）
  例如："负荷中等·双通道协同良好" "负荷偏高·冗余原则违反"

> **填充原则**：
> 1. 左列写"传达了什么"——锚定 MODULE 2 的信息单元清单
> 2. 中列写"用什么策略传达"——锚定 MODULE 3 的认知弧线 + MODULE 1 的动效策略
> 3. 右列写"观众脑中发生了什么变化"——锚定 MODULE 6 的认知负荷 + MODULE 3 的知识状态追踪
> 4. 如果某段落的认知建构尚未完成（如前几段仅在铺垫概念，核心模型未成型），
>    右列卡片的主标题写"（基础蓄能中）"，副标题写"模型尚未闭合"

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **信息类型转换点**：从概念引入切到数据展示，或从数据切到机制解释——信息类型的质变是最强切分信号
2. **认知负荷断裂点**：信息密度骤然变化（密→疏 或 疏→密），通常伴随视觉留白或节奏变化
3. **通道模式切换点**：从"旁白+图形"切到"纯数据动画无旁白"，或从"概念图解"切到"实例演示"
4. **视觉风格/色调转换点**：色彩系统切换、动效风格变化、构图模式改变
5. **旁白论证转折点**：从陈述切到提问，从"是什么"切到"为什么"，从解释切到总结

典型的信息动画段落划分：

| 信息动画子类型 | 典型段落数 | 典型结构 |
|-------------|-----------|---------|
| 30s 社交媒体数据动画 | 3 段 | 钩子数据 → 核心对比 → 结论CTA |
| 60–90s 概念解释动画 | 4 段 | 问题提出 → 概念定义 → 机制演示 → 整合总结 |
| 2–3min 数据驱动叙事 | 4–5 段 | 现象引入 → 数据全景 → 关键发现 → 因果解释 → 结论与启示 |
| 3–5min 深度科普/教育动画 | 5–6 段 | 钩子 → 背景铺垫 → 核心概念 → 证据展示 → 机制拆解 → 认知闭合 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：5 段 → 50 + 5×90 + 20 = 520
示例：3 段 → 50 + 3×90 + 20 = 340
示例：6 段 → 50 + 6×90 + 20 = 610

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为双向——认知建构方向严格为 左→中→右（信息→编码→认知效果）
- 禁止在右列"心智建构"卡片中写画面描述——右列只写观众的认知状态变化
- 禁止在左列"信息输入"卡片中写设计手法——左列只写被传达的信息内容本身
