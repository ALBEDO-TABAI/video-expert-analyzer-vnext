## SVG 生活影像双契约结构图 — AGENT PROMPT

### 任务描述

根据你完成的 Cinematic Vlog 分析报告，生成一张
**生活素材（日常锚点）/ 影像策略（升格操作）/ 观众契约（亲密↔审美）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是 Cinematic Vlog 的**双契约管线**：
每一行代表 Vlog 的一个段落，三列分别回答——
- 左列：这一段从日常生活中**选取**了什么素材？（空间/事件/时刻）
- 中列：这一段用什么**影像策略**对日常素材进行了升格处理？
- 右列：这些策略最终在观众端完成了什么样的**契约效果**？（亲密感 or 审美感 or 兼得）

> **三栏锚定说明**：
> - 左列锚定 MODULE 5（空间系统）+ MODULE 4（时间诗学·可报告性筛选）
> - 中列锚定 MODULE 1（视觉语法）+ MODULE 3（声音建筑学）+ MODULE 6（多轨交织·审美跃迁点）
> - 右列锚定 MODULE 2（自我呈现系统）+ MODULE 7（开场契约/收尾姿态）

> **箭头方向论证**：水平箭头为**单向（→）**。
> 与 MV 图的双向对照关系不同，Cinematic Vlog 的核心操作是一条
> **升格管线**——日常素材经过影像策略的加工，产出特定的观众契约效果。
> 这是一条因果流（素材 → 处理 → 效果），不是平行互注。
> 观众不会先"感受契约效果"再反向推导"用了什么素材"——
> 因果方向严格为 左→中→右。

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  生活素材（日常锚点） |  影像策略（升格操作） |  观众契约（亲密↔审美）
[分隔线]
[段落 1 行]  左卡片  →  中卡片  →  右卡片
[段落 2 行]  左卡片  →  中卡片  →  右卡片
...
[段落 N 行]  左卡片  →  中卡片  →  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）

### 颜色分配规则

每一个段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**该段落的主导契约模式**——即这一段主要在服务真实性契约、审美契约，
还是两者的某种张力状态。按以下逻辑选色：

| 段落契约模式 | 推荐色 | 逻辑依据 |
|---|---|---|
| 开场契约 / 观看邀请建立 | c-coral | 暖色开场能量，建立第一次接触的陪伴/审美预期 |
| 日常浸入 / 常规亲密 | c-teal | 冷静、舒适的陪伴节奏，低强度持续型亲密感 |
| 审美跃迁 / 诗意升格 | c-purple | 高密度视觉诗意时刻，审美契约的集中兑现 |
| 情感暴露 / 后台泄露 | c-pink | 温暖的脆弱感，真实性契约的关键兑现点 |
| 双契约合流 / 亲密与审美同时达到高点 | c-green | 两条契约同时履行 = Cinematic Vlog 的理想态 |
| 收尾姿态 / 契约结算 | c-gray | 克制退场，从影像世界回到创作者-观众关系 |
| 可根据实际段落的契约张力灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各段落）。

### 连接箭头规格

**水平箭头**（同行，左→中→右，表示升格管线方向）：

```svg
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图**：水平箭头在 MV 图中是"对照关系"（画面 ↔ 主题 ↔ 语言），
> 但在 Cinematic Vlog 图中是**升格管线**——日常素材经过影像策略加工，
> 产出观众端的契约效果。因此箭头方向严格为 左→中→右（单向），不可反向。
> 这条管线的核心问题是：同一段日常素材，经过不同的影像策略处理后，
> 会产出"亲密感优先"还是"审美感优先"还是"两者兼得"的不同契约效果。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示 Vlog 时间线的推进与累积。
> - 左列的垂直箭头 = 生活场景/空间的切换与叠加（策展路径）
> - 中列的垂直箭头 = 影像策略的演变（如从手持跟拍逐渐切向 Gimbal 诗意段）
> - 右列的垂直箭头 = 观众关系的深化（从初次接触到契约兑现到告别）

### 内容填充规则

根据你的 Cinematic Vlog 分析报告，将 Vlog 的时间轴划分为 3–6 个段落
（不是机械等分时间，而是按**契约模式的切换**来切分）。
为每个段落填写三张卡片：

**左卡片（生活素材 — 日常锚点）**
- 主标题：该段落的核心生活场景/空间/事件（≤8字）
  例如："晨光厨房" "雨天散步" "咖啡店独处" "夜间书桌"
- 副标题：该场景在身份定位中的功能（≤14字）
  例如："同一性·日常可共鸣" "差异性·独特生活方式"

**中卡片（影像策略 — 升格操作）**
- 主标题：该段落的主导影像策略名称（≤8字）
  例如："手持跟拍" "Gimbal诗意段" "慢镜蒙太奇" "固定机位独白"
- 副标题：时间戳范围，格式 "MM:SS – MM:SS"

**右卡片（观众契约 — 亲密↔审美）**
- 主标题：该段落完成的契约效果（≤10字）
  例如："亲密感·日常陪伴" "审美感·视觉诗意" "双契约·亲密中的美"
- 副标题：对应的理论操作标注（≤14字）
  例如："PSI邀请·Horton模拟在场" "诗意模式·Nichols审美凝视"
  "后台泄露·Goffman特权访问" "校准业余·Abidin真实性信号"

> **填充原则**：
> 1. 左列写"选了什么日常素材"——锚定 MODULE 5 的空间策展 + MODULE 4 的可报告性筛选
> 2. 中列写"用了什么影像手段升格"——锚定 MODULE 1 的视觉语法 + MODULE 3 的声音建筑 + MODULE 6 的审美跃迁点
> 3. 右列写"观众最终获得什么体验"——锚定 MODULE 2 的自我呈现分析 + MODULE 7 的契约判定
> 4. 如果某段落的影像策略是"刻意不升格"（如保留原始手持、不配BGM），
>    中列卡片的主标题写"（去升格）"，副标题写时间戳，
>    右列卡片写"真实性优先·裸素材信号"——
>    因为"不加工"本身也是一种升格管线的选择（Abidin 的校准式业余感）

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **审美跃迁点**：影像策略从日常记录突然切换至电影化语言的那一刻（如从手持切到 Gimbal 慢镜），必须是一个切分点
2. **契约模式切换**：从"亲密感建构"切换到"审美升格"，或反向切换
3. **空间/场景转换**：从一个生活空间进入另一个（如从卧室到街道、从室内到户外）
4. **声音层断裂**：BGM 首次进入/退出/切换，旁白从有到无或从无到有
5. **PSI 模式切换**：创作者从"不看镜头"切到"直视镜头对观众说话"，或反之
6. **时间操控切换**：从实时记录切到延时/升格慢镜/蒙太奇段落

典型的 Cinematic Vlog 段落划分：

| Vlog 子类型 | 典型段落数 | 典型结构 |
|------------|-----------|---------|
| 日常记录型（短） | 3–4 段 | 晨间开场 → 日常浸入 → 审美点缀 → 夜间收尾 |
| 日常记录型（长） | 5–6 段 | 开场契约 → 日常浸入A → 审美跃迁 → 日常浸入B → 反思独白 → 告别收尾 |
| 旅行/体验型 | 4–5 段 | 出发/入场 → 空间探索 → 审美高潮（核心景观） → 日常穿插 → 回归收尾 |
| 氛围/情绪型 | 3–4 段 | 诗意开场 → 情绪积累 → 审美顶点 → 余韵收束 |
| 叙事/散文型 | 4–5 段 | 主题提出 → 生活证据A → 生活证据B → 反思升华 → 开放收尾 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：5 段 → 50 + 5×90 + 20 = 520
示例：3 段 → 50 + 3×90 + 20 = 340
示例：6 段 → 50 + 6×90 + 20 = 610

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为双向——升格管线方向严格为 左→中→右
- 禁止在右列"观众契约"卡片中写画面描述——右列只写观众获得了什么契约体验
- 禁止在中列"影像策略"卡片中写内容描述——中列只写用了什么影像技术手段
