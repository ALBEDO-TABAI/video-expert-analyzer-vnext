## SVG 梗/病毒内容传播结构图 — AGENT PROMPT

### 任务描述

根据你完成的梗/病毒内容分析报告，生成一张
**模板骨架（结构层）/ 不协调引擎（机制层）/ 传播着陆（效果层）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是 Shifman（2014）+ Nissenbaum & Shifman（2018）的
**模板传播因果链**：
每一行代表梗的一个意义建构阶段，三列分别回答——
- 左列：这个阶段的**模板结构**是什么？固定层如何框定可变层？
- 中列：这个阶段的**不协调/幽默机制**如何运作？认知落差从哪里产生？
- 右列：这个机制最终**着陆**在什么传播效果上？驱动了什么分享行为？

> **三栏锚定声明**：
> 左列锚定 MODULE 1（模板结构解剖）——固定层/可变层的结构映射
> 中列锚定 MODULE 3（不协调结构与幽默机制）——setup/punchline 的认知落差
> 右列锚定 MODULE 6 + MODULE 2.3（传播动力学 + 立场维度）——为什么值得分享

> **箭头方向论证**：
> 水平箭头为**单向（左→中→右）**。
> 理由：梗的意义生产是一条因果链——模板结构**使能**不协调机制的运作，
> 不协调机制**产生**传播驱动力。结构是前提，机制是过程，传播是结果。
> 这与 MV 图的"双向对照"不同，因为梗不是三条平行线的互注，
> 而是一条"结构→效果"的因果流。

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  模板骨架（结构层） |  不协调引擎（机制层） |  传播着陆（效果层）
[分隔线]
[阶段 1 行]  左卡片  →  中卡片  →  右卡片
[阶段 2 行]  左卡片  →  中卡片  →  右卡片
...
[阶段 N 行]  左卡片  →  中卡片  →  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每阶段行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 阶段的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际阶段数调整，确保最后一行底边 + 20px = viewBox 高度）

### 卡片规格

每个卡片为双行文字卡片：

```svg
<g class="node c-{COLOR}">
  <rect x="{X}" y="{Y}" width="168" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="{X+84}" y="{Y+19}"
        text-anchor="middle" dominant-baseline="central">
    主标题（≤10字）
  </text>
  <text class="ts" x="{X+84}" y="{Y+39}"
        text-anchor="middle" dominant-baseline="central">
    副标题（≤16字）
  </text>
</g>
```

卡片点击绑定 sendPrompt，内容为"详细分析 [主标题] 阶段"：
```svg
<g class="node c-{COLOR}" onclick="sendPrompt('详细分析 [主标题] 阶段')">
```

### 颜色分配规则

每一个意义建构阶段分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**梗的意义建构功能阶段**，而非视觉装饰。按以下逻辑选色：

| 意义建构阶段特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| 模板激活/首帧钩子/格式识别 | c-coral | 第一眼接触——暖色标记"入口能量" |
| 预期铺垫/情境建立/setup | c-teal | 冷静地构建认知基线——等待被打破 |
| 不协调爆破/punchline/翻转 | c-purple | 最高密度认知冲击——意义碰撞的爆发点 |
| 互文激活/文化门槛/圈层信号 | c-pink | 温度上升——"懂的人才懂"的共谋感 |
| 传播触发/立场着陆/分享冲动 | c-green | 从"看"到"转"的行动转化——价值"变现" |
| 模板开放/再创作邀请/变异预留 | c-gray | 克制收尾——结构让渡给下一个创作者 |
| 可根据实际梗结构灵活调整 | — | — |

禁止对不同阶段使用相同颜色（颜色必须区分各意义建构阶段）。

### 连接箭头规格

**水平箭头**（同行，左→中→右，表示因果流向）：

```svg
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图**：水平箭头严格为**单向（左→中→右）**。
> 这与 MV 图的"对照关系"（画面 ↔ 主题 ↔ 语言）不同。
> 梗的意义生产是一条因果链：模板结构**使能**→ 不协调机制**运转**→ 传播效果**产生**。
> 如果某个阶段的不协调机制尚未激活（如 setup 阶段），
> 中列卡片写"（蓄能中）"，右列写"预期基线建立中"。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示意义建构的阶段推进。
> 左列的垂直箭头 = 模板结构的逐层暴露（从格式识别到细节解码）；
> 中列的垂直箭头 = 不协调张力的积累与释放（setup → punchline）；
> 右列的垂直箭头 = 传播冲动的蓄积与爆发（从旁观到分享到再创作）。

### 内容填充规则

根据你的梗/病毒内容分析报告，将梗的意义建构过程划分为 3–6 个阶段
（不是机械等分时间，而是按**意义建构功能的转换**来切分）。
为每个阶段填写三张卡片：

**左卡片（模板骨架 — 结构层）**
- 主标题：该阶段激活的核心模板元素名称（≤8字）
  例如："底图格式识别" "上格·拒绝姿态" "可变文字位" "音频模板钩子"
- 副标题：该元素属于固定层还是可变层，及其结构功能（≤14字）
  例如："固定层·情感框架锚定" "可变层·语义负载入口"

**中卡片（不协调引擎 — 机制层）**
- 主标题：该阶段的不协调/幽默机制类型（≤8字）
  例如："预期建立" "语域不协调" "比例反转" "元层反讽"
- 副标题：具体描述认知落差的构建方式（≤14字）
  例如："正式画面×口语文案" "微事→史诗级修辞"
  如果该阶段尚未触发不协调，副标题写"（蓄能中）·认知基线"

**右卡片（传播着陆 — 效果层）**
- 主标题：该阶段产生的传播效果/分享动机（≤10字）
  例如："社交货币充值" "圈层身份信号" "立场表演完成" "再创作邀请"
- 副标题：对应 Shifman 三维度或 Jenkins 传播性框架（≤14字）
  例如："stance驱动·态度共鸣" "form驱动·模板复用"
  如果该阶段尚未产生传播效果，主标题写"（蓄能中）"

> **填充原则**：
> 1. 左列写"什么结构在这个阶段被激活"——锚定 MODULE 1 模板解剖
> 2. 中列写"这个阶段的不协调机制如何运作"——锚定 MODULE 3 幽默机制
> 3. 右列写"受众因此产生了什么传播行为倾向"——锚定 MODULE 6 传播动力学
> 4. 如果某阶段处于"铺垫"状态（结构已激活但不协调尚未爆破），
>    中列写"（蓄能中）"，右列写"预期基线建立中"——
>    这种"蓄能"本身是传播结构的必要组成部分

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **不协调爆破点（punchline/twist）**：认知落差最大的时刻，必须是一个切分点
2. **模板层切换点**：从固定层进入可变层、或从视觉模态进入文字模态的转换
3. **互文激活点**：梗开始依赖特定前置知识才能理解的时刻
4. **模态切换点**：从无声到有声、从静态到动态、从画面到文字的转换
5. **传播触发点**：受众从"看"转为"想分享"的决策瞬间（通常紧随 punchline）

典型的梗结构阶段划分：

| 梗类型 | 典型阶段数 | 典型结构 |
|--------|-----------|---------|
| 图像宏（Image Macro） | 3 阶段 | 模板识别 → 上格铺垫 → 下格翻转 |
| 短视频梗（< 15s） | 3–4 阶段 | 首帧钩子 → 情境铺垫 → 翻转爆破 → 余味/再创作邀请 |
| POV/剧情类视频梗 | 4–5 阶段 | 字幕框架 → 场景建立 → 预期积累 → 翻转 → 传播触发 |
| 混剪/Remix/Edit | 4–5 阶段 | 素材识别 → 互文激活 → 并置碰撞 → 新意义涌现 → 圈层信号着陆 |
| 行为梗/Challenge | 3–4 阶段 | 动作模板展示 → 个人化执行 → 效果对比/意外 → 模仿邀请 |
| Copypasta/文字梗 | 3 阶段 | 格式/风格识别 → 内容-风格不协调 → 复制粘贴冲动 |

### viewBox 高度计算公式

```
阶段数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：3 阶段 → 50 + 3×90 + 20 = 340
示例：4 阶段 → 50 + 4×90 + 20 = 410
示例：5 阶段 → 50 + 5×90 + 20 = 520

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同阶段设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为双向——因果流向严格为 左→中→右
- 禁止在右列"传播着陆"卡片中写画面描述——右列只写传播效果与动机
- 禁止将行标签写为"第 N 段"——行标签应体现该阶段的意义建构功能
- 禁止使用"病毒式传播"一词——使用"传播性（spreadability）"或"扩散"
