## SVG 口播修辞结构图 — AGENT PROMPT

### 任务描述

根据你完成的口播/讲述视频分析报告，生成一张
**身体表演（Actio）/ 话语功能（修辞段落）/ 可信度着陆（Ethos 效果）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是**表演-话语-可信度的因果管线**：
每一行代表口播的一个修辞段落，三列分别回答——
- 左列：这一段讲述者的**身体/声音表演**在做什么？（怎么说的）
- 中列：这一段的**话语功能**是什么？处于论证弧线的哪个阶段？（说了什么）
- 右列：这些表演和话语共同为讲述者**建构了什么可信度效果**？（产生了什么 Ethos）

> **三栏锚定说明**：
> 左列锚定 MODULE 1（身体修辞）+ MODULE 2（声音修辞）的分析发现；
> 中列锚定 MODULE 4（话语结构拆解）的段落功能判定；
> 右列锚定 MODULE 6（Ethos 建构策略）的三重修辞诉求盘点。

---

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

---

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  身体表演（Actio） |  话语功能（修辞段落） |  可信度着陆（Ethos效果）
[分隔线]
[段落 1 行]  左卡片  →  中卡片  →  右卡片
[段落 2 行]  左卡片  →  中卡片  →  右卡片
...
[段落 N 行]  左卡片  →  中卡片  →  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）

### 卡片规格

每个卡片为双行文字卡片：

```svg
<g class="node c-{COLOR}">
  <rect x="{X}" y="{Y}" width="168" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="{X+84}" y="{Y+19}"
        text-anchor="middle" dominant-baseline="central">
    主标题（≤10字）
  </text>
  <text class="ts" x="{X+84}" y="{Y+39}"
        text-anchor="middle" dominant-baseline="central">
    副标题（≤16字）
  </text>
</g>
```

卡片点击绑定 sendPrompt，内容为"详细分析 [主标题] 段落"：
```svg
<g class="node c-{COLOR}" onclick="sendPrompt('详细分析 [主标题] 段落')">
```

### 颜色分配规则

每一个修辞段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**口播话语功能的修辞阶段**，而非视觉装饰。按以下逻辑选色：

| 话语功能阶段特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| 开钩/注意力捕获/痛点戳刺 | c-coral | 高能量开场，需要暖色突破滑动流 |
| 背景铺垫/问题定义/语境建立 | c-teal | 冷静的信息充能阶段，建立理性基调 |
| 核心论点/观点输出/方案展示 | c-purple | 高密度论证集中区，修辞重心所在 |
| 证据展示/案例叙述/经验佐证 | c-pink | 情感与逻辑交织，Pathos 升温区 |
| 升华/价值观提炼/金句输出 | c-green | 意义凝结，从具体到抽象的跃迁 |
| CTA/收束/行动号召/悬念留钩 | c-gray | 克制收尾，从论述退出到行动指令 |
| 可根据实际话语结构灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各修辞段落）。

### 连接箭头规格

**水平箭头**（同行，左→中→右，表示表演-话语-可信度的因果流向）：

```svg
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图与箭头方向论证**：
> 水平箭头严格为 左→中→右（单向），不可反向或画为双向。
> 理由：口播分析的核心操作是一条**因果管线**——
> 身体表演（Actio）为话语内容提供**传达载体**，
> 话语内容又为 Ethos 建构提供**可信度素材**。
> 这与 MV 图中画面-主题-歌词的"平行对照"关系不同，
> 口播中"怎么说的"服务于"说了什么"，
> "说了什么"又服务于"让观众相信什么"——
> 这是 Goffman 框架下"表演→印象管理→定义情境"的单向逻辑。
> 如果画为双向，将模糊这条因果链条的方向性。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示时间/论证进程的推进。
> 左列的垂直箭头 = 身体表演能量与策略的演变（如从克制到激昂）；
> 中列的垂直箭头 = 话语功能的论证递进（如从铺垫到论点到收束）；
> 右列的垂直箭头 = 可信度的累积/转化（如从 Logos 权威到 Pathos 共鸣到 Ethos 锁定）。

### 内容填充规则

根据你的口播分析报告，将口播的时间轴划分为 3–6 个修辞段落
（不是机械等分时间，而是按**话语功能的转换**来切分）。
为每个段落填写三张卡片：

**左卡片（身体表演 — Actio）**
- 主标题：该段落的主导表演策略标签（≤8字）
  例如："凝视锚定" "自嘲微笑" "手势密集" "能量爆发" "停顿蓄力" "目光偏移·回忆"
- 副标题：景别 + 语速/语调变化的简要描述（≤14字）
  例如："近景·语速加快·音调上扬" "中景·停顿3秒·压低声线"

**中卡片（话语功能 — 修辞段落）**
- 主标题：该段落在论证弧线中的功能角色（≤8字）
  例如："开钩·痛点" "背景定义" "核心论点" "案例佐证" "金句升华" "CTA收束"
- 副标题：时间戳范围，格式 "MM:SS – MM:SS"

**右卡片（可信度着陆 — Ethos 效果）**
- 主标题：该段落为讲述者建构的可信度效果（≤10字）
  例如："建构型Ethos·专业性" "Pathos·共鸣锚定" "经验性权威确立" "亲和力·人设锁定"
- 副标题：对应 Aristotle 三重诉求中的主导诉求路径（≤14字）
  例如："Logos主导·数据论证建立信任" "Pathos主导·自我暴露拉近距离"

> **填充原则**：
> 1. 左列写"身体和声音在这一段做了什么"——锚定 MODULE 1+2 的表演分析
> 2. 中列写"这一段在论证弧线中干什么"——锚定 MODULE 4 的话语结构
> 3. 右列写"讲述者因此获得了什么可信度"——锚定 MODULE 6 的 Ethos 盘点
> 4. 如果某段落的身体表演与话语内容存在不一致（MODULE 5 识别的"真实性泄露"），
>    左列卡片的主标题标注 "⚡" 前缀，例如 "⚡表情与词矛盾"

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **修辞诉求主导权切换**：从 Logos 主导切到 Pathos 主导（或反之），必须是一个切分点
2. **能量/语速断裂点**：语速突变、音量骤升/骤降、出现显著停顿（≥2秒）
3. **话语功能转换点**：从铺垫切到论点、从论点切到案例、从案例切到总结
4. **目光/身体姿态突变**：目光从直视切到偏移、身体从静态切到大幅手势
5. **叠加层介入点**：B-roll 插入、花字出现、画面分割等视觉断裂

典型的口播段落划分：

| 口播子类型 | 典型段落数 | 典型结构 |
|----------|-----------|---------|
| 15–30s 短口播（抖音/Reels） | 3 段 | 开钩 → 核心观点 → CTA |
| 1–3min 知识分享 | 4–5 段 | 开钩 → 问题定义 → 核心论点 → 证据/案例 → 总结CTA |
| 3–5min 深度观点 | 5–6 段 | 开钩 → 背景 → 论点1 → 论点2/反驳 → 升华 → CTA |
| 5–10min 长讲述/故事 | 5–6 段 | 开钩/悬念 → 背景 → 发展 → 高潮/转折 → 反思 → 收束 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：5 段 → 50 + 5×90 + 20 = 520
示例：3 段 → 50 + 3×90 + 20 = 340
示例：6 段 → 50 + 6×90 + 20 = 610

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为双向——因果管线方向严格为 左→中→右（表演→话语→可信度）
- 禁止在右列"可信度着陆"卡片中写画面描述——右列只写讲述者获得了什么可信度效果
- 禁止在左列"身体表演"卡片中写话语内容——左列只写身体/声音层面的表演策略
