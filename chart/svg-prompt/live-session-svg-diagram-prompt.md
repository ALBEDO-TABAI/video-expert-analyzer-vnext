## SVG 现场演出临场性结构图 — AGENT PROMPT

### 任务描述

根据你完成的现场演出影像分析报告，生成一张
**现场事件（临场性信号源）/ 中介策略（覆盖与处理）/ 临场性传递（屏幕效果）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是 Auslander（2008）/ Phelan（1993）的**临场性中介悖论**：
每一行代表演出的一个能量段落，三列分别回答——
- 左列：这一段的现场**实际发生了什么**？哪些临场性信号存在于物理空间中？
- 中列：镜头/剪辑/混音**如何处理**了这些信号？采取了什么覆盖和中介策略？
- 右列：经过中介后，屏幕观众**接收到了什么**程度的临场感？最终的在场效果是什么？

> **三栏锚定声明**：
> - 左列锚定 MODULE 2（临场性信号清单）+ MODULE 6（身体与声音真实性）
> - 中列锚定 MODULE 1（覆盖策略与视觉语法）+ MODULE 2（中介化干预清单）
> - 右列锚定 MODULE 2（临场性策略判定）+ MODULE 4（能量弧线·综合能量）

> **水平箭头方向论证**：
> 水平箭头严格为 **左→中→右（单向）**，不可反向或双向。
> 理由：现场演出影像的信息流是单向的因果链——
> 物理空间中的临场性信号（左）被摄影/导播/混音的中介策略（中）选择性捕捉和处理，
> 最终转化为屏幕观众可感知的临场性效果（右）。
> 这不是 MV 中画面与歌词的"互相注释"关系，
> 而是"原始信号 → 中介处理 → 传递效果"的不可逆管线。
> 每一步都存在信息损耗或信息增强——这正是分析的核心关注点。

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  现场事件（信号源） |  中介策略（覆盖与处理） |  临场性传递（屏幕效果）
[分隔线]
[段落 1 行]  左卡片  →  中卡片  →  右卡片
[段落 2 行]  左卡片  →  中卡片  →  右卡片
...
[段落 N 行]  左卡片  →  中卡片  →  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）

### 卡片规格

每个卡片为双行文字卡片：

```svg
<g class="node c-{COLOR}">
  <rect x="{X}" y="{Y}" width="168" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="{X+84}" y="{Y+19}"
        text-anchor="middle" dominant-baseline="central">
    主标题（≤10字）
  </text>
  <text class="ts" x="{X+84}" y="{Y+39}"
        text-anchor="middle" dominant-baseline="central">
    副标题（≤16字）
  </text>
</g>
```

卡片点击绑定 sendPrompt，内容为"详细分析 [主标题] 段落"：
```svg
<g class="node c-{COLOR}" onclick="sendPrompt('详细分析 [主标题] 段落')">
```

### 颜色分配规则

每一个能量段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**能量弧线的阶段特征**，而非视觉装饰。按以下逻辑选色：

| 能量阶段特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| 入场/仪式开启/能量蓄积 | c-coral | 暖色低温启动，标记"门槛进入" |
| 预热/节奏建立/场域校准 | c-teal | 冷静的能量充能阶段，场域尚在建构中 |
| 攀升/观众卷入/反馈回路激活 | c-pink | 温度上升，集体情感开始涌现 |
| 高峰/集体欢腾/能量爆发 | c-purple | 最高密度的临场性——阈限体验的顶点 |
| 释放/余韵/能量回落 | c-green | 高峰后的"着陆"，仪式开始收束 |
| 收束/退场/仪式关闭 | c-gray | 克制收尾，从阈限空间回到日常 |
| 可根据实际能量弧线灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各能量段落）。

### 连接箭头规格

**水平箭头**（同行，左→中→右，表示临场性中介传递方向）：

```svg
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图**：水平箭头表示**临场性的中介传递管线**——
> 现场信号（左）经中介策略（中）处理后，传递为屏幕效果（右）。
> 箭头方向严格为 左→中→右（单向），不可反向。
> 这与 MV 图中画面-主题-语言的"互相注释"双向箭头不同：
> 现场演出影像的信息流是不可逆的——
> 物理空间中的事件无法被屏幕反向影响，中介处理也无法被撤回。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示时间/能量弧线的推进。
> 左列的垂直箭头 = 现场事件的时间推进与临场性信号的演变（身体疲劳累积、观众情绪升级、空间状态变化）；
> 中列的垂直箭头 = 中介策略的适应性变化（导播如何随演出能量调整覆盖策略）；
> 右列的垂直箭头 = 屏幕临场性效果的累积——观众"在场感"是否随时间加深。

### 内容填充规则

根据你的现场演出影像分析报告，将演出时间轴划分为 3–6 个能量段落
（不是机械等分时间，而是按**能量弧线的阶段转换**来切分）。
为每个段落填写三张卡片：

**左卡片（现场事件 — 临场性信号源）**
- 主标题：该段落的核心现场事件/表演状态（≤8字）
  例如："乐队热身/调音" "全场合唱爆发" "独奏即兴段" "安可仪式"
- 副标题：最显著的临场性信号类型（≤14字）
  例如："观众呼喊·身体密度高" "汗水可见·声音微颤"

**中卡片（中介策略 — 覆盖与处理）**
- 主标题：该段落的主导中介策略（≤8字）
  例如："长镜头·手持跟拍" "快切多机位" "慢镜回放插入" "固定全景覆盖"
- 副标题：时间戳范围，格式 "MM:SS – MM:SS"

**右卡片（临场性传递 — 屏幕效果）**
- 主标题：该段落传递给屏幕观众的在场效果（≤10字）
  例如："沉浸式在场模拟" "能量峰值传递" "仪式感建构" "情感余韵延伸"
- 副标题：临场性传递的成功度评估（≤14字）
  例如："高：反馈回路可见" "中：空间感保留，互动缺失"

> **填充原则**：
> 1. 左列写"现场发生了什么"——锚定 MODULE 2 的临场性信号清单 + MODULE 6 的身体/声音真实性
> 2. 中列写"镜头/剪辑/混音如何处理"——锚定 MODULE 1 的覆盖策略 + MODULE 2 的中介化干预清单
> 3. 右列写"屏幕观众最终感受到什么"——锚定 MODULE 2 的临场性策略判定 + MODULE 4 的综合能量评估
> 4. 如果某段落的中介策略导致临场性信号被严重削弱或扭曲（如过度修音、
>    大量慢镜头破坏时间连续性），右列卡片的主标题写"临场性断裂"，
>    副标题写具体的损耗原因（如"修音痕迹破坏声学真实"）

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **能量断裂点**：能量密度的显著跳变（从低能到高能的陡升，或高峰后的骤降）
2. **歌曲/曲目边界**：如果分镜表记录了多首歌曲，歌曲间的过渡是天然切分点
3. **集体行为转折**：观众行为模式的显著变化（从静听到合唱、从站立到律动）
4. **中介策略切换**：导播/摄影策略的明显转变（如从多机位快切突然进入长镜头）
5. **MC/互动段落**：表演者与观众的直接语言互动，标志"音乐流"的暂停
6. **仪式边界**：开场仪式（灯暗/intro）和收束仪式（安可/鞠躬/散场）

典型的现场演出段落划分：

| 演出类型 | 典型段落数 | 典型结构 |
|----------|-----------|---------|
| 单曲 Live Session（3-5min） | 3 段 | 入场/建立 → 表演核心 → 收束/余韵 |
| Studio Session（如 Tiny Desk） | 3–4 段 | 空间入场 → 首曲·场域建立 → 中段·能量攀升 → 末曲·情感高点 |
| 完整演唱会片段（10-20min） | 4–5 段 | 开场蓄能 → 预热 → 攀升 → 高峰爆发 → 收束 |
| 完整演唱会（60min+） | 5–6 段 | 仪式开启 → 能量建立 → 第一高峰 → 低谷/亲密段 → 终极高峰 → 安可/仪式关闭 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：5 段 → 50 + 5×90 + 20 = 520
示例：3 段 → 50 + 3×90 + 20 = 340
示例：6 段 → 50 + 6×90 + 20 = 610

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为双向——临场性中介传递方向严格为 左→中→右
- 禁止在右列"临场性传递"卡片中写画面描述——右列只写屏幕观众最终接收到的在场效果
- 禁止将左列写成镜头语言描述——左列只写现场物理空间中实际发生的事件和信号，与镜头无关
