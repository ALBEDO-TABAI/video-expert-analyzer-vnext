## SVG 广告修辞结构图 — AGENT PROMPT

### 任务描述

根据你完成的广告/品牌短片分析报告，生成一张
**参照世界（意义源）/ 说服弧线（修辞功能）/ 品牌着陆（意义终点）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是 Williamson（1978）的**意义转移管线**：
每一行代表广告的一个修辞段落，三列分别回答——
- 左列：这一段从哪个文化意义库中**借用**了什么意义？
- 中列：这一段执行的**说服功能**是什么？处于说服弧线的哪个阶段？
- 右列：这些意义最终**着陆**在品牌/产品的什么层面上？

---

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

---

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  参照世界（意义源） |  说服弧线（修辞功能） |  品牌着陆（意义终点）
[分隔线]
[段落 1 行]  左卡片  ←→  中卡片  ←→  右卡片
[段落 2 行]  左卡片  ←→  中卡片  ←→  右卡片
...
[段落 N 行]  左卡片  ←→  中卡片  ←→  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）

### 卡片规格

每个卡片为双行文字卡片：

```svg
<g class="node c-{COLOR}">
  <rect x="{X}" y="{Y}" width="168" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="{X+84}" y="{Y+19}"
        text-anchor="middle" dominant-baseline="central">
    主标题（≤10字）
  </text>
  <text class="ts" x="{X+84}" y="{Y+39}"
        text-anchor="middle" dominant-baseline="central">
    副标题（≤16字）
  </text>
</g>
```

卡片点击绑定 sendPrompt，内容为"详细分析 [主标题] 段落"：
```svg
<g class="node c-{COLOR}" onclick="sendPrompt('详细分析 [主标题] 段落')">
```

### 颜色分配规则

每一个修辞段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**说服阶段的修辞功能**，而非视觉装饰。按以下逻辑选色：

| 说服阶段特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| 钩子/注意力捕获/世界入口 | c-coral | 高能量开场，需要暖色吸引力 |
| 参照世界建立/氛围铺陈 | c-teal | 冷静的意义充能阶段 |
| 情感升级/痛点放大/张力积累 | c-pink | 情绪的粉色温度上升 |
| 冲突顶点/情感爆发/转折 | c-purple | 高密度修辞集中点 |
| 品牌揭示/意义转移完成 | c-green | 意义着陆 = 价值"变现" |
| CTA/收束/价值观宣言 | c-gray | 克制收尾，从叙事退出到行动指令 |
| 可根据实际说服弧线灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各修辞段落）。

### 连接箭头规格

**水平箭头**（同行，左→中→右，表示意义转移方向）：

```svg
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图**：水平箭头在 MV 图中是"对照关系"（画面 ↔ 主题 ↔ 语言），
> 但在广告图中是**因果流向**——意义从参照世界流向修辞功能，再流向品牌。
> 因此箭头方向严格为 左→中→右（单向），不可反向。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示时间/说服进程的推进。
> 左列的垂直箭头 = 参照系统的切换/叠加；
> 中列的垂直箭头 = 说服阶段的递进；
> 右列的垂直箭头 = 品牌含义的累积/深化。

### 内容填充规则

根据你的广告分析报告，将广告的时间轴划分为 3–6 个修辞段落
（不是机械等分时间，而是按**说服功能的转换**来切分）。
为每个段落填写三张卡片：

**左卡片（参照世界 — 意义源）**
- 主标题：该段落调用的核心参照系统名称（≤8字）
  例如："自然/黎明" "都市独居" "运动员精神" "母女代际"
- 副标题：该参照系统携带的文化含义（≤14字）
  例如："纯净·新生·未被污染" "独立·自主·微孤独"

**中卡片（说服弧线 — 修辞功能）**
- 主标题：该段落在说服弧线中的功能角色（≤8字）
  例如："钩子/世界入口" "情感卷入" "痛点放大" "品牌揭示" "CTA收束"
- 副标题：时间戳范围，格式 "MM:SS – MM:SS"

**右卡片（品牌着陆 — 意义终点）**
- 主标题：该段落为品牌完成的意义着陆（≤10字）
  例如："品牌=自然之力" "品牌=理解你的人" "产品=解决方案"
- 副标题：对应 Keller 权益层级或 Aristotle 诉求路径（≤14字）
  例如："含义层·品牌意境建设" "回应层·pathos 情感共鸣"

> **填充原则**：
> 1. 左列写"从哪里借来了什么"——锚定 MODULE 2 的参照系统分析
> 2. 中列写"这一段在说服弧线中干什么"——锚定 MODULE 3 的弧线识别
> 3. 右列写"品牌因此获得了什么"——锚定 MODULE 6 的战略解码
> 4. 如果某段落尚未完成意义转移（如前几段仅在铺陈参照世界，品牌未出场），
>    右列卡片的主标题写"（蓄能中）"，副标题写"意义尚未着陆"

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **品牌揭示时刻**：品牌/产品首次明确出现的那一帧，必须是一个切分点
2. **情绪/色调断裂点**：音乐风格突变、色调系统切换、剪辑节奏陡变
3. **参照系统切换点**：画面从一个参照世界进入另一个（如从自然场景切到城市）
4. **VO/语言轨的论证转折**：旁白从陈述切到提问，或从问题切到答案
5. **CTA/标语出现**：行动号召或品牌标语出现，标志说服弧线收束

典型的广告段落划分：

| 广告类型 | 典型段落数 | 典型结构 |
|----------|-----------|---------|
| 15s 信息流广告 | 3 段 | 钩子 → 产品演示 → CTA |
| 30s 效果广告 | 4 段 | 钩子 → 痛点 → 解决方案 → CTA |
| 60s 品牌形象片 | 4–5 段 | 世界建构 → 情感卷入 → 转折 → 品牌揭示 → 标语收束 |
| 2–3min Campaign 主片 | 5–6 段 | 入口 → 参照铺陈 → 情感升级 → 高潮/转折 → 品牌着陆 → 价值观宣言 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：5 段 → 50 + 5×90 + 20 = 520
示例：3 段 → 50 + 3×90 + 20 = 340
示例：6 段 → 50 + 6×90 + 20 = 610

---

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为双向——意义转移方向严格为 左→中→右
- 禁止在右列"品牌着陆"卡片中写画面描述——右列只写品牌获得了什么意义
