## SVG 节奏混剪声画同步结构图 — AGENT PROMPT

### 任务描述

根据你完成的节奏混剪分析报告，生成一张
**音乐骨架（节奏源）/ 剪辑操作（同步机制）/ 感官效果（体验终点）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是 Chion（1994）的**声画同步效应链**：
每一行代表混剪的一个能量段落（由音乐结构划分），三列分别回答——
- 左列：这一段的**音乐层**提供了什么节奏骨架和能量信号？
- 中列：剪辑师在这一段执行了什么**同步操作**？beat-cut 策略、技法密度、ASL 如何？
- 右列：这些操作在观众感知中制造了什么**体验效果**？

> **三栏锚定说明**：
> 左列锚定 MODULE 1（节拍骨架解剖）的音乐结构分段与 MODULE 4（声画对位）的音乐轨；
> 中列锚定 MODULE 1（beat-cut 对位精度）、MODULE 3（视觉语法统计）、MODULE 6（技术指纹）；
> 右列锚定 MODULE 5（能量弧线与高潮工程学）、MODULE 7（开钩与收尾）。

> **箭头方向论证**：水平箭头严格为 **左→中→右（单向）**。
> 节奏混剪的创造性因果链是：音乐的节奏结构（源）→ 决定剪辑师的同步操作选择（机制）→ 产生观众端的感官体验（效果）。
> 音乐是主人，画面是仆人——这不是 MV 式的"画面与歌词互相注释"的平行关系，
> 而是"音乐驱动剪辑，剪辑制造效果"的单向因果流。

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

---

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  音乐骨架（节奏源） |  剪辑操作（同步机制） |  感官效果（体验终点）
[分隔线]
[段落 1 行]  左卡片  →  中卡片  →  右卡片
[段落 2 行]  左卡片  →  中卡片  →  右卡片
...
[段落 N 行]  左卡片  →  中卡片  →  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）

### 卡片规格

每个卡片为双行文字卡片：

```svg
<g class="node c-{COLOR}">
  <rect x="{X}" y="{Y}" width="168" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="{X+84}" y="{Y+19}"
        text-anchor="middle" dominant-baseline="central">
    主标题（≤10字）
  </text>
  <text class="ts" x="{X+84}" y="{Y+39}"
        text-anchor="middle" dominant-baseline="central">
    副标题（≤16字）
  </text>
</g>
```

卡片点击绑定 sendPrompt，内容为"详细分析 [主标题] 段落"：
```svg
<g class="node c-{COLOR}" onclick="sendPrompt('详细分析 [主标题] 段落')">
```

### 颜色分配规则

每一个能量段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**能量阶段的节奏功能**，而非视觉装饰。按以下逻辑选色：

| 能量阶段特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| Intro/Hook：注意力捕获，节奏承诺建立 | c-coral | 开场能量点火，暖色标记"起跑" |
| Verse/Build-up：素材展示，节奏稳态，能量蓄积 | c-teal | 冷静的充能阶段，节奏尚在铺底 |
| Pre-drop：张力递增，ASL 压缩，闪切加密 | c-pink | 能量温度持续攀升，接近沸点 |
| Drop/Climax：节奏爆发，synchresis 峰值，overtonal 冲击 | c-purple | 最高密度修辞集中点，感官过载 |
| Breakdown/Breath：能量骤降，长镜头呼吸，情感切换 | c-gray | 克制与留白，张力循环的"谷底" |
| Outro/收尾：能量衰减或硬停，节奏承诺兑现 | c-green | 完成感，体验弧线着陆 |
| 可根据实际音乐结构灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各能量段落）。

### 连接箭头规格

**水平箭头**（同行，左→中→右，表示声画因果链方向）：

```svg
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图**：水平箭头在 MV 图中是"对照关系"（画面 ↔ 主题 ↔ 语言），
> 但在节奏混剪图中是**因果驱动链**——音乐节奏驱动剪辑操作，剪辑操作制造感官效果。
> 因此箭头方向严格为 左→中→右（单向），不可反向。
> 这与广告图的"意义转移管线"方向一致，但驱动力不同：
> 广告的驱动力是"意义从参照世界流向品牌"；
> 混剪的驱动力是"节奏从音乐流向身体感知"。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示能量进程的推进。
> 左列的垂直箭头 = 音乐结构段落的推进（intro→verse→drop→…）；
> 中列的垂直箭头 = 剪辑策略的变化与递进（从克制到爆发）；
> 右列的垂直箭头 = 观众体验强度的累积与起伏（张力-释放循环）。

### 内容填充规则

根据你的节奏混剪分析报告，将混剪的时间轴按**音乐结构的段落转换**划分为 3–6 个能量段落。
为每个段落填写三张卡片：

**左卡片（音乐骨架 — 节奏源）**
- 主标题：该段落的音乐结构名称 + 能量关键词（≤8字）
  例如："Verse·蓄压" "Drop·爆发" "Breakdown·呼吸" "Intro·点火"
- 副标题：BPM + 主导节奏元素 + 能量级（≤14字）
  例如："128BPM·hi-hat 16分·能量7" "bass drop进入·能量10"

**中卡片（剪辑操作 — 同步机制）**
- 主标题：该段落的核心剪辑策略名称（≤8字）
  例如："度量加速" "on-beat 硬切" "呼吸长镜头" "闪切轰炸" "速度斜坡"
- 副标题：ASL + on-beat率 + 技法密度 或时间戳（≤14字）
  例如："ASL 0.5s·on-beat 92%·4技法" "00:32 – 00:48"

**右卡片（感官效果 — 体验终点）**
- 主标题：该段落在观众端制造的核心体验（≤10字）
  例如："节奏锁定·信任建立" "生理紧迫感" "overtonal 冲击" "张力卸压·情感切入" "体验着陆"
- 副标题：对应的理论机制或能量评分（≤14字）
  例如："synchresis 峰值·能量10" "Pearlman 张力释放" "Eisenstein 度量加速"

> **填充原则**：
> 1. 左列写"音乐在这一段提供了什么节奏骨架"——锚定 MODULE 1 的音乐结构分段
> 2. 中列写"剪辑师如何回应这个骨架"——锚定 MODULE 1 的 beat-cut 分析 + MODULE 3 视觉语法 + MODULE 6 技法
> 3. 右列写"观众因此获得了什么体验"——锚定 MODULE 5 的能量弧线
> 4. 如果某段落处于蓄压阶段（如 verse），右列的主标题写感知效果而非"（蓄能中）"——
>    因为混剪的每一段都在制造体验，不存在"意义尚未着陆"的情况（与广告不同）

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **Drop 时刻**：bass/低频进入的那一帧，能量骤变，必须是一个切分点
2. **Breakdown 入口**：音乐密度骤降、旋律/人声独立出现的时刻
3. **BPM 或节拍型变化点**：节奏型从 4/4 切到 3/4，或 BPM 发生阶梯变化
4. **素材来源切换点**：从一组素材来源切到另一组（如从动画切到实拍）
5. **剪辑策略断裂点**：ASL 骤变、转场类型系统切换、技法密度突变
6. **音乐段落边界**：verse/chorus/bridge 等常规音乐结构边界

典型的节奏混剪段落划分：

| 混剪类型 | 典型段落数 | 典型结构 |
|----------|-----------|---------|
| 30–60s 短混剪 | 3 段 | Build-up → Drop → Outro |
| 1–2min 标准混剪（单 drop） | 4 段 | Intro → Verse/Build → Drop → Outro |
| 2–3min 标准混剪（双 drop） | 5–6 段 | Intro → Build → Drop 1 → Breakdown → Drop 2 → Outro |
| 3min+ 长混剪 / Showcase | 5–6 段 | 多循环结构，按张力-释放循环数确定段落 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：5 段 → 50 + 5×90 + 20 = 520
示例：3 段 → 50 + 3×90 + 20 = 340
示例：6 段 → 50 + 6×90 + 20 = 610

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为双向——因果驱动方向严格为 左→中→右
- 禁止在右列"感官效果"卡片中写画面描述——右列只写观众端的体验/感知效果
- 禁止在左列"音乐骨架"卡片中写视觉信息——左列只写音乐/声音参数
