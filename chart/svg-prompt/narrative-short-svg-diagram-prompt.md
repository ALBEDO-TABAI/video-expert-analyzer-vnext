## SVG 叙事短片结构图 — AGENT PROMPT

### 任务描述

根据你完成的叙事短片分镜表分析报告，生成一张
**场面呈现（视听证据）/ 叙事经济（功能定位）/ 省略与暗示（意义压缩）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是叙事短片的**三层透视**：
每一行代表短片的一个叙事段落，三列分别回答——
- 左列：这一段**呈现了什么**？（看得见/听得见的视听证据）
- 中列：这一段在叙事结构中**承担什么功能**？（角色时刻/信息释放/蓄力/收束）
- 右列：这一段**省略或暗示了什么**？（被压缩进道具/沉默/空间/缺席中的意义）

三栏之间是**互相注释的诊断关系**——
你无法理解"省略了什么"，除非先看到"呈现了什么"；
你也无法完全理解"呈现了什么"，除非认识到"什么被刻意留在了画面之外"。
中列的叙事功能是连接表层与深层的解释性桥梁。

---

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

---

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  场面呈现（视听证据） |  叙事经济（功能定位） |  省略与暗示（意义压缩）
[分隔线]
[段落 1 行]  左卡片  ←→  中卡片  ←→  右卡片
[段落 2 行]  左卡片  ←→  中卡片  ←→  右卡片
...
[段落 N 行]  左卡片  ←→  中卡片  ←→  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）

### 卡片规格

每个卡片为双行文字卡片：

```svg
<g class="node c-{COLOR}">
  <rect x="{X}" y="{Y}" width="168" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="{X+84}" y="{Y+19}"
        text-anchor="middle" dominant-baseline="central">
    主标题（≤10字）
  </text>
  <text class="ts" x="{X+84}" y="{Y+39}"
        text-anchor="middle" dominant-baseline="central">
    副标题（≤16字）
  </text>
</g>
```

卡片点击绑定 sendPrompt，内容为"详细分析 [主标题] 段落"：
```svg
<g class="node c-{COLOR}" onclick="sendPrompt('详细分析 [主标题] 段落')">
```

### 颜色分配规则

每一个叙事段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**叙事段落在短片结构中的功能角色**，而非视觉装饰。按以下逻辑选色：

| 叙事功能特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| 情境建立/日常锚定/角色聚焦 | c-coral | 暖色建立观众与角色的初始连接 |
| 信息蓄力/细节积累/环境浸入 | c-teal | 冷静的观察阶段，叙事密度在沉默中升高 |
| 压力递增/张力积累/不安浮现 | c-pink | 情绪温度上升，省略策略开始制造悬念 |
| 角色时刻/抉择瞬间/行为爆发 | c-purple | 叙事密度最高点，角色做出改变处境的抉择 |
| 顿悟爆发/意义浮现/认知翻转 | c-green | 被压缩的意义在此刻释放，观众的理解跃迁 |
| 收束/处境定格/开放悬置 | c-gray | 克制收尾，叙事从"讲述"退出到"沉默" |
| 可根据实际叙事段落功能灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各叙事段落）。

### 连接箭头规格

**水平箭头**（同行，双向，表示互相注释关系）：

```svg
<!-- 左→中 -->
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<!-- 中→左 -->
<line x1="254" y1="{卡片中心Y+6}" x2="208" y2="{卡片中心Y+6}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<!-- 中→右 -->
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<!-- 右→中 -->
<line x1="470" y1="{卡片中心Y+6}" x2="424" y2="{卡片中心Y+6}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图**：水平箭头为**双向（↔）**。
> 这是叙事短片分析与广告分析的根本区别——
> 广告的意义转移是单向因果流（来源→机制→品牌），
> 但短片的"呈现"与"省略"是**互构关系**：
> 你需要看到左列"呈现了什么"才能理解右列"省略了什么"，
> 同时你需要意识到右列"省略了什么"才能真正读懂左列"呈现的为什么是这些而非其他"。
> 中列（叙事功能）作为诊断桥梁，双向连接两端。
> 箭头画为双向以体现这一诊断性互文关系。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示叙事进程的推进。
> 左列的垂直箭头 = 视听证据的时序递进（场面如何一步步展开）；
> 中列的垂直箭头 = 叙事功能的阶段递进（从建立→蓄力→爆发→收束）；
> 右列的垂直箭头 = 省略/暗示的层层叠加（被压缩的意义如何逐步积累直到释放）。

### 内容填充规则

根据你的叙事短片分析报告，将短片的时间轴划分为 3–6 个叙事段落
（不是机械等分时间，而是按**叙事情境转换点或信息释放节奏断裂点**来切分）。
为每个段落填写三张卡片：

**左卡片（场面呈现 — 视听证据）**
- 主标题：该段落的核心视觉场景或声音事件（≤8字）
  例如："空走廊·钟声" "厨房·沉默晚餐" "雨中公车站"
- 副标题：主导视觉/听觉特征（景别、运镜、色调、关键音效）（≤14字）
  例如："特写·手持·低饱和" "全景·固定机位·环境音"

> **填充锚定**：左列内容来自 MODULE 1（视觉语法）+ MODULE 4（声画关系）的分析数据

**中卡片（叙事经济 — 功能定位）**
- 主标题：该段落在叙事结构中的功能角色（≤8字）
  例如："情境锚定" "张力蓄力" "角色时刻" "顿悟释放" "处境定格"
- 副标题：时间戳范围，格式 "MM:SS – MM:SS"

> **填充锚定**：中列内容来自 MODULE 3（角色时刻定位）+ MODULE 7（结构弧线）的分析判断

**右卡片（省略与暗示 — 意义压缩）**
- 主标题：该段落省略或暗示的核心内容（≤10字）
  例如："前史完全缺席" "动机未言明" "结局开放悬置" "物件承载潜台词"
- 副标题：省略/暗示通过什么载体实现（≤14字）
  例如："咖啡温度变化=时间流逝" "信封未打开=拒绝面对" "空椅子=缺席的人"

> **填充锚定**：右列内容来自 MODULE 2（省略策略）+ MODULE 6（符号系统）的分析发现

> **填充原则**：
> 1. 左列写"银幕上发生了什么"——基于可观察的视听事实
> 2. 中列写"这一段在叙事经济中干什么"——基于结构性功能判断
> 3. 右列写"什么被压缩进了画面的缝隙中"——基于省略策略和符号分析
> 4. 如果某段落的省略尚未可辨识（如开场阶段信息量不足以判断省略了什么），
>    右列卡片的主标题写"（待显影）"，副标题写"省略尚未构成可读模式"

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **角色时刻/抉择点**：角色做出改变处境的行为或拒绝行为的那一刻，必须是一个切分点
2. **顿悟时刻**：认知/情感的突然揭示发生的位置（如果存在）
3. **信息释放断裂**：叙事信息密度突然升高或骤降的时刻
4. **声画关系突变**：从对白模式切到沉默，或从配乐切到环境音
5. **空间转换点**：场景空间的物理切换（尤其在空间数量有限的短片中，每次空间切换都是重大事件）
6. **色调/光影断裂**：画面的视觉系统发生显著变化

典型的叙事短片段落划分：

| 短片子类型 | 典型段落数 | 典型结构 |
|----------|-----------|---------|
| 情境型短片（3-7min） | 3–4 段 | 日常锚定 → 微小异变 → 角色时刻 → 处境定格 |
| 叙事型短片（10-25min） | 4–6 段 | 情境建立 → 信息蓄力 → 张力积累 → 角色时刻 → 顿悟/翻转 → 收束 |
| 诗性/实验型短片 | 3–5 段 | 情绪基调建立 → 变奏/叠加 → 节奏断裂 → 形式回归/消解 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：4 段 → 50 + 4×90 + 20 = 430
示例：5 段 → 50 + 5×90 + 20 = 520
示例：6 段 → 50 + 6×90 + 20 = 610

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为单向——"呈现"与"省略"是互构关系，箭头严格为双向（↔）
- 禁止在右列"省略与暗示"卡片中写画面描述——右列只写"看不见的东西"
- 禁止将叙事功能（中列）写成画面描述的重复——中列是结构性判断，不是内容概述
