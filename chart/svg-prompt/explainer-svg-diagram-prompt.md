## SVG 教学视频认知脚手架结构图 — AGENT PROMPT

### 任务描述

根据你完成的讲解/教学视频分析报告，生成一张
**视觉通道（认知素材）/ 教学操作（脚手架功能）/ 认知建构（学习成果）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是 Mayer（2009）的**多媒体学习认知理论**与 Gagné（1965）的**教学事件序列**的交叉映射：
每一行代表教学视频的一个**知识脚手架段落**，三列分别回答——
- 左列：这一段在视觉通道上**投入**了什么认知素材？（动画/图表/隐喻/实拍……）
- 中列：这一段执行的**教学操作**是什么？处于 Gagné 教学事件序列的哪个阶段？
- 右列：学习者因此应当获得什么**认知建构**？（新概念/关联/心理模型/迁移能力……）

> **三栏锚定声明**：
> 左列锚定 MODULE 1（认知负荷审计·视觉通道扫描）+ MODULE 5（概念可视化与隐喻系统）
> 中列锚定 MODULE 3（教学事件序列·Gagné 映射）+ MODULE 2（知识建筑·脚手架路径）
> 右列锚定 MODULE 2（知识建筑·概念清单）+ MODULE 7（讲解者教学在场·认知确认点）

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  视觉通道（认知素材） |  教学操作（脚手架功能） |  认知建构（学习成果）
[分隔线]
[段落 1 行]  左卡片  →  中卡片  →  右卡片
[段落 2 行]  左卡片  →  中卡片  →  右卡片
...
[段落 N 行]  左卡片  →  中卡片  →  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）


### 颜色分配规则

每一个知识脚手架段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**教学功能阶段**，而非视觉装饰。按以下逻辑选色：

| 教学阶段特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| 注意力获取/动机建立/开场钩子 | c-coral | 高能量唤醒阶段，暖色标记启动信号 |
| 先备知识激活/术语预训练 | c-teal | 冷静的地基建设阶段，为新知识做认知准备 |
| 核心概念呈现/主干知识讲解 | c-purple | 认知密度最高的教学核心段，深色标记重点区 |
| 学习指导/类比展开/隐喻可视化 | c-pink | 温暖的脚手架支撑阶段，帮助编码与理解 |
| 练习引导/总结回顾/知识巩固 | c-green | 知识着陆 = 概念从工作记忆编码进长期记忆 |
| 迁移提示/行动号召/收束 | c-gray | 克制收尾，从学习语境退出到应用语境 |
| 可根据实际教学弧线灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各教学阶段）。

### 连接箭头规格

**水平箭头**（同行，左→中→右，表示认知脚手架因果方向）：

```svg
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图**：水平箭头表示**认知脚手架的因果流向**——
> 视觉素材（左列）被教学操作（中列）组织和加工，最终产出学习者的认知建构（右列）。
> 这是 Mayer CTML 的核心逻辑：素材输入 → 选择/组织/整合 → 心理模型。
> 因此箭头方向严格为 **左→中→右（单向）**，不可反向或双向。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示教学进程的推进。
> 左列的垂直箭头 = 视觉素材的复杂度递进（从简单图示到复杂动画/隐喻）；
> 中列的垂直箭头 = 教学事件的阶段递进（从注意力获取到迁移促进）；
> 右列的垂直箭头 = 概念建构的层级递进（从基础概念到核心概念到扩展概念）。

### 内容填充规则

根据你的教学视频分析报告，将视频的时间轴划分为 3–6 个知识脚手架段落
（不是机械等分时间，而是按**教学功能的转换**来切分）。
为每个段落填写三张卡片：

**左卡片（视觉通道 — 认知素材）**
- 主标题：该段落投入的核心视觉素材类型（≤8字）
  例如："问题场景动画" "流程图解" "水管类比动画" "代码实操录屏" "对比表格"
- 副标题：视觉通道负荷评估 + 双编码状态（≤14字）
  例如："低负荷·双编码对齐" "高负荷·文字冗余违规" "中负荷·隐喻准确"

**中卡片（教学操作 — 脚手架功能）**
- 主标题：该段落在教学事件序列中的功能角色（≤8字）
  例如："获取注意力" "激活先备知识" "呈现核心概念" "类比引导理解" "总结与迁移"
- 副标题：时间戳范围，格式 "MM:SS – MM:SS"

**右卡片（认知建构 — 学习成果）**
- 主标题：该段落应为学习者建构的认知成果（≤10字）
  例如："建立学习动机" "激活已知→锚定新知" "形成X的心理模型" "概念迁移至新情境"
- 副标题：对应 Bloom 认知层级或 Gagné 事件编号（≤14字）
  例如："理解层·Gagné事件4-5" "应用层·Gagné事件6" "（缺失·事件6未覆盖）"

> **填充原则**：
> 1. 左列写"视觉通道投入了什么"——锚定 MODULE 1 的通道负荷扫描 + MODULE 5 的可视化清单
> 2. 中列写"这一段在教学进程中干什么"——锚定 MODULE 3 的 Gagné 事件映射
> 3. 右列写"学习者因此获得了什么"——锚定 MODULE 2 的概念清单与脚手架路径
> 4. 如果某段落的教学操作缺失（如未提供学习指导、未做总结），
>    右列卡片的主标题写"（脚手架缺口）"，副标题写"事件X未覆盖·编码风险"

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **核心概念引入时刻**：第一个新术语/新概念出现的那一帧，必须是一个切分点
2. **教学功能转换点**：从"注意力获取"切到"概念讲解"，或从"讲解"切到"类比/举例"
3. **认知负荷陡变点**：信息密度突然升高或降低、视觉复杂度突变
4. **隐喻/可视化切换点**：从一种视觉表征方式切到另一种（如从流程图切到实物动画）
5. **旁白语气转折点**：从陈述切到提问，从讲解切到总结，从学术语气切到对话语气
6. **分段/总结标记**：旁白中出现"接下来""总结一下""回顾一下"等元认知信号

典型的教学视频段落划分：

| 教学视频类型 | 典型段落数 | 典型结构 |
|----------|-----------|---------|
| 1-3min 微课/知识胶囊 | 3 段 | 钩子 → 核心概念 → 总结行动 |
| 3-5min 概念讲解 | 4 段 | 动机建立 → 预训练 → 概念+类比 → 总结迁移 |
| 5-10min 标准教学视频 | 4–5 段 | 钩子 → 先备激活 → 核心讲解 → 学习指导(类比/举例) → 总结+行动 |
| 10-20min 深度教程 | 5–6 段 | 动机 → 预训练 → 概念A → 类比/练习A → 概念B → 综合总结+迁移 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：5 段 → 50 + 5×90 + 20 = 520
示例：3 段 → 50 + 3×90 + 20 = 340
示例：6 段 → 50 + 6×90 + 20 = 610

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为双向——认知脚手架因果方向严格为 左→中→右
- 禁止在右列"认知建构"卡片中写画面描述——右列只写学习者获得了什么认知成果
- 禁止在左列"视觉通道"卡片中写教学评价——左列只描述视觉素材本身
