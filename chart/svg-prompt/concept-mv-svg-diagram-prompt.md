## SVG 概念 MV 感知建筑图 — AGENT PROMPT

### 任务描述

根据你完成的概念 MV 分析报告，生成一张
**视觉场域（感官素材）/ 体验建构（感知状态）/ 声画博弈（多媒体关系）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是概念 MV 的**三维感知收敛模型**：
每一行代表一个体验段落，三列分别回答——
- 左列：这一段的**视觉场域**在做什么？（空间/色调/运镜/材质——MODULE 1 + MODULE 3 的感官素材层）
- 中列：这一段正在建构什么**感知体验**？（组织逻辑类型 + 感知强度 + 主导体验模式——MODULE 2 + MODULE 6 的体验建构层）
- 右列：这一段的视觉轨与音乐轨之间形成了怎样的**声画关系**？（Cook 三元模型判定——MODULE 4 的多媒体关系层）

三列的关系是**相互注释**而非因果流向——
视觉场域塑造感知体验，感知体验反过来框定我们如何阅读视觉场域；
声画关系影响体验建构，体验语境也重新定义声画关系的感知效果。
因此水平箭头为**双向**。

> **三栏锚定声明**：
> 左列锚定 MODULE 1（视觉语法与节奏拓扑）+ MODULE 3（视觉空间系统）
> 中列锚定 MODULE 2（组织逻辑识别）+ MODULE 6（体验弧线与感知建筑）
> 右列锚定 MODULE 4（多轨关系分析：声画博弈）

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  视觉场域（感官素材） |  体验建构（感知状态） |  声画博弈（多媒体关系）
[分隔线]
[段落 1 行]  左卡片  ←→  中卡片  ←→  右卡片
[段落 2 行]  左卡片  ←→  中卡片  ←→  右卡片
...
[段落 N 行]  左卡片  ←→  中卡片  ←→  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）

### 卡片规格

每个卡片为双行文字卡片：

```svg
<g class="node c-{COLOR}">
  <rect x="{X}" y="{Y}" width="168" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="{X+84}" y="{Y+19}"
        text-anchor="middle" dominant-baseline="central">
    主标题（≤10字）
  </text>
  <text class="ts" x="{X+84}" y="{Y+39}"
        text-anchor="middle" dominant-baseline="central">
    副标题（≤16字）
  </text>
</g>
```

卡片点击绑定 sendPrompt，内容为"详细分析 [主标题] 段落"：
```svg
<g class="node c-{COLOR}" onclick="sendPrompt('详细分析 [主标题] 段落')">
```

### 颜色分配规则

每一个体验段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**主导感知模式**，而非视觉装饰。按以下逻辑选色：

| 感知模式特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| 催眠/冥想/低刺激沉浸 | c-teal | 冷静的深层沉浸，水面般的平稳 |
| 建构/铺陈/感知契约建立 | c-coral | 暖色的入口温度，世界正在展开 |
| 困惑/陌生化/认知摩擦 | c-purple | 高密度的感知压力，不可解的紫色 |
| 冲击/爆发/感知过载 | c-pink | 感官强度的粉色炽热 |
| 恍惚/漂浮/时间扭曲 | c-gray | 感知边界的模糊，灰色地带 |
| 收束/消散/余响/闭合 | c-green | 体验的着陆与释放 |
| 可根据实际体验弧线灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各体验段落）。

### 连接箭头规格

**水平箭头**（同行，左↔中，中↔右，表示相互注释关系）：

水平箭头为**双向**——每对卡片之间画两条单向箭头构成双向：

```svg
<!-- 左→中 -->
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<!-- 中→左 -->
<line x1="254" y1="{卡片中心Y+6}" x2="208" y2="{卡片中心Y+6}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>

<!-- 中→右 -->
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<!-- 右→中 -->
<line x1="470" y1="{卡片中心Y+6}" x2="424" y2="{卡片中心Y+6}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图**：水平箭头在广告图中是"因果流向"（左→中→右，单向），
> 但在概念 MV 图中是**相互注释**——视觉场域与感知体验之间、
> 感知体验与声画关系之间，不存在单向的因果关系，
> 而是三个维度**共同涌现**出一个不可分割的感知事件。
> 因此箭头方向必须为双向。
>
> **箭头方向论证**：概念 MV 的分析核心是 Cook（1998）的多媒体关系理论，
> 该理论的核心前提正是：多媒体意义产生于媒介之间的**隙缝**（gap），
> 而非任何单一媒介的单向投射。双向箭头在视觉上表达了这一理论前提。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示体验段落的时间推进。
> 左列的垂直箭头 = 视觉场域的变形/切换/叠加；
> 中列的垂直箭头 = 感知体验的强度变化与模式转换；
> 右列的垂直箭头 = 声画关系类型的动态演化（如从 conformance 滑向 contest）。

### 内容填充规则

根据你的概念 MV 分析报告，将 MV 的时间轴划分为 3–6 个体验段落
（不是机械等分时间，而是按**感知模式的转换**来切分）。
为每个段落填写三张卡片：

**左卡片（视觉场域 — 感官素材）**
- 主标题：该段落的主导视觉空间/材质标签（≤8字）
  使用感知性标签而非叙事性标签——
  例如："白色虚空" "液态金属面" "颗粒化肉身" "霓虹脉冲"
  而非 "房间" "街道" "舞台"
- 副标题：主导色调 + 运镜模式的组合标签（≤14字）
  例如："冷蓝灰·匀速缓推" "高饱和暖·手持震颤" "黑白·静止凝视"

**中卡片（体验建构 — 感知状态）**
- 主标题：该段落的主导组织逻辑 + 感知模式（≤10字）
  使用 MODULE 2 的类型表术语——
  例如："关联逻辑·催眠" "累积逻辑·冲击" "梦工作·困惑" "变奏逻辑·恍惚"
- 副标题：时间戳范围 + 感知强度标注，格式 "MM:SS–MM:SS [N/10]"
  例如："00:00–01:15 [3/10]" "01:16–02:40 [8/10]"

**右卡片（声画博弈 — 多媒体关系）**
- 主标题：该段落的 Cook 模型判定（≤10字）
  必须使用三元模型术语——
  例如："Conformance" "Complementation" "Contest" "Contest→Conform"
  如果段落内发生关系类型转换，用箭头标注方向
- 副标题：声画关系的感知效果一句话描述（≤14字）
  例如："视听一体·沉浸加深" "速度矛盾·身体撕裂" "画面独立·音乐退场"

> **填充原则**：
> 1. 左列写"感官层面发生了什么"——锚定 MODULE 1 的视觉语法 + MODULE 3 的空间场域
> 2. 中列写"这一段在建构什么感知体验"——锚定 MODULE 2 的组织逻辑 + MODULE 6 的体验弧线
> 3. 右列写"视觉和音乐之间在玩什么游戏"——锚定 MODULE 4 的多轨关系分析
> 4. 如果某段落的声画关系难以判定（如视觉与音乐各自运行于完全独立的逻辑层），
>    右列卡片的主标题写"平行独立"，副标题写"两轨各自为政·零交叉"
> 5. 禁止在左列写叙事内容（概念 MV 没有叙事）——左列只写感官素材的物理特征
> 6. 禁止在右列写音乐内容本身——右列只写视觉与音乐之间的**关系**

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **声画关系类型转换点**：Cook 模型判定从一种类型切换到另一种的瞬间（如 conformance → contest），这几乎总是概念 MV 最重要的结构节点
2. **组织逻辑切换点**：视觉的组织原理发生变化（如从关联逻辑切换到累积逻辑）
3. **感知强度跃迁点**：感知强度发生不可渐变的阶跃（非缓慢过渡，而是突然的量级变化）
4. **视觉空间/材质断裂点**：画面从一个视觉场域硬切到另一个完全不同的场域
5. **音乐段落结构边界**：与 verse/chorus/bridge/outro 的边界重合的画面变化

典型的概念 MV 段落划分：

| 概念 MV 子类型 | 典型段落数 | 典型体验结构 |
|---------------|-----------|-------------|
| 抽象概念 MV（纯色彩/形状/运动） | 3–4 段 | 建构 → 发展 → 高潮 → 消散 |
| 超现实概念 MV（可辨认但非逻辑的人物/空间） | 4–5 段 | 日常入口 → 陌生化 → 梦深层 → 冲击/爆发 → 余响 |
| 舞蹈概念 MV（编舞为核心载体） | 3–5 段 | 身体静止 → 渐入 → 全力 → 变奏 → 收束 |
| 叙事残影概念 MV（被拆解的叙事碎片） | 4–6 段 | 碎片抛出 → 累积 → 伪因果 → 断裂 → 悬置 → 回响 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：5 段 → 50 + 5×90 + 20 = 520
示例：3 段 → 50 + 3×90 + 20 = 340
示例：6 段 → 50 + 6×90 + 20 = 610

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为单向——概念 MV 的三维度是**相互注释**关系，箭头必须双向
- 禁止在左列"视觉场域"卡片中写叙事内容——概念 MV 没有叙事，左列只写感官素材的物理特征（色调/材质/运镜/空间类型）
- 禁止在中列"体验建构"卡片中使用叙事术语（如"主角觉醒""冲突升级"）——应使用感知模式术语（如"催眠""冲击""恍惚""困惑"）
- 禁止在右列"声画博弈"卡片中写音乐本身的描述——右列只写视觉与音乐之间的**关系类型**及其感知效果
