## SVG 预告片修辞结构图 — AGENT PROMPT

### 任务描述

根据你完成的预告片/Teaser/Trailer 分析报告，生成一张
**展示切片（What is Shown）/ 修辞弧线（Rhetorical Arc）/ 观众心理效果（Audience Effect）**
三栏对照的 SVG 结构图。

这张图的底层逻辑是 **Hediger（2001）的"诱惑机制"与 Kernan（2004）的修辞说服模型**：
每一行代表预告片的一个修辞段落，三列分别回答——
- 左列：这一段从正片中**选择展示**了什么碎片/切片？（信息经济学的"揭示"侧）
- 中列：这一段在修辞弧线中执行的**说服功能**是什么？
- 右列：这些展示试图在观众心中制造什么**认知/情感效果**？

> **三栏关系：单向因果流**
> 展示什么 → 执行什么修辞功能 → 产生什么观众效果
> 水平箭头方向严格为 左→中→右。
> 理由：预告片是一个**目的导向的修辞文本**——每一个展示选择都服务于特定的说服目标，
> 每一个说服目标都指向特定的观众心理效果。这是因果流，不是平行对照。

> **模块锚定声明**：
> 左列锚定 MODULE 2（信息经济学）+ MODULE 3（世界切片）
> 中列锚定 MODULE 6（结构弧线）
> 右列锚定 MODULE 6.3（预设读者/观众效果）+ MODULE 2.2（谜面清单）

---

### 图表规格

- 格式：SVG，viewBox="0 0 680 [动态高度]"
- 宽度：width="100%"，不得写死像素值
- 无外层容器 div，直接输出 `<svg>` 元素
- 背景：透明（不设任何背景色）
- 必须包含 `role="img"`、`<title>` 和 `<desc>` 无障碍标签

---

### 结构规则

**整体布局：三栏 + 标题行**

```
[标题行]  展示切片（画面碎片） |  修辞弧线（说服功能） |  观众效果（心理目标）
[分隔线]
[段落 1 行]  左卡片  →  中卡片  →  右卡片
[段落 2 行]  左卡片  →  中卡片  →  右卡片
...
[段落 N 行]  左卡片  →  中卡片  →  右卡片
```

**坐标系**
- 安全区：x=40 到 x=640，y=40 到 y=(viewBox高度-20)
- 三列 x 起点：左列 x=40，中列 x=256，右列 x=472
- 每列卡片宽度：168px
- 列间连接箭头区域：x=208→254（左到中），x=424→470（中到右）

**行间距计算**
- 标题行高度：约 35px（含分隔线）
- 每段落行高度：56px（卡片高）+ 22px（行间距）= 78px
- 第 N 段落的卡片 y 起点 = 50 + (N-1) × 90
  （根据实际段落数调整，确保最后一行底边 + 20px = viewBox 高度）

### 卡片规格

每个卡片为双行文字卡片：

```svg
<g class="node c-{COLOR}">
  <rect x="{X}" y="{Y}" width="168" height="56" rx="8" stroke-width="0.5"/>
  <text class="th" x="{X+84}" y="{Y+19}"
        text-anchor="middle" dominant-baseline="central">
    主标题（≤10字）
  </text>
  <text class="ts" x="{X+84}" y="{Y+39}"
        text-anchor="middle" dominant-baseline="central">
    副标题（≤16字）
  </text>
</g>
```

卡片点击绑定 sendPrompt，内容为"详细分析 [主标题] 段落"：
```svg
<g class="node c-{COLOR}" onclick="sendPrompt('详细分析 [主标题] 段落')">
```

### 颜色分配规则

每一个修辞段落分配一个颜色，同一行的三列卡片使用**同一颜色**。
颜色编码**预告片修辞阶段的说服功能**，而非视觉装饰。按以下逻辑选色：

| 修辞阶段特征 | 推荐色 | 逻辑依据 |
|---|---|---|
| 开钩/注意力捕获/类型锚定 | c-coral | 预告片必须在前几秒"抓住"观众——高能量暖色标记入口 |
| 世界建立/设定展示/氛围铺陈 | c-teal | 冷静的世界观充能阶段，展示"你将进入什么样的世界" |
| 角色引入/明星展示/认同建立 | c-pink | 温暖的人物连接阶段，建立情感锚点 |
| 冲突升级/威胁揭示/张力积累 | c-purple | 高密度信息释放与情绪加压 |
| 奇观爆发/高潮蒙太奇/节奏顶点 | c-green | 视觉承诺的"兑现"——展示正片最壮观的画面资产 |
| 品牌锁定/标题卡/上映信息/尾刺 | c-gray | 从修辞叙事退出到信息传达，克制收尾 |
| 可根据实际修辞弧线灵活调整 | — | — |

禁止对不同段落使用相同颜色（颜色必须区分各修辞段落）。

### 连接箭头规格

**水平箭头**（同行，左→中→右，表示"展示→功能→效果"的因果方向）：

```svg
<line x1="208" y1="{卡片中心Y}" x2="254" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
<line x1="424" y1="{卡片中心Y}" x2="470" y2="{卡片中心Y}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

> **设计意图**：水平箭头在 MV 图中是"对照关系"（画面 ↔ 主题 ↔ 语言），
> 但在预告片图中是**因果流向**——展示选择驱动修辞功能，修辞功能指向观众效果。
> 因此箭头方向严格为 左→中→右（单向），不可反向。
> 这与广告图的箭头逻辑（意义转移方向）相似，但语义不同：
> 广告的因果流是"意义从参照世界流向品牌"，
> 预告片的因果流是"碎片经过修辞编排流向观众心理"。

**垂直箭头**（同列，上段→下段，三列各一条）：

```svg
<line x1="{列中心X}" y1="{上段卡片底边+2}" x2="{列中心X}" y2="{下段卡片顶边-2}"
      class="arr" marker-end="url(#arrow)"
      stroke="var(--color-border-secondary)"/>
```

三列中心 X 值：左列 124，中列 340，右列 556。

> **设计意图**：垂直箭头表示修辞进程的推进。
> 左列的垂直箭头 = 展示碎片的递进/升级（从安全到危险，从日常到奇观）
> 中列的垂直箭头 = 修辞阶段的递进（从建立到积累到爆发到锁定）
> 右列的垂直箭头 = 观众心理效果的累积/升级（从好奇到紧张到震撼到决策）

### 内容填充规则

根据你的预告片分析报告，将预告片的时间轴划分为 3–6 个修辞段落
（不是机械等分时间，而是按**修辞功能的转换**来切分）。
为每个段落填写三张卡片：

**左卡片（展示切片 — 画面碎片）**
- 主标题：该段落展示的核心画面内容/空间/人物（≤8字）
  例如："废墟城市远景" "角色A特写" "追逐/爆炸蒙太奇" "黑暗中的剪影"
- 副标题：该展示的信息释放程度（≤14字）
  例如："世界观全貌·高透明度" "人物仅剪影·身份遮蔽" "快切碎片·信息过载"

**中卡片（修辞弧线 — 说服功能）**
- 主标题：该段落在修辞弧线中的功能角色（≤8字）
  例如："开钩/类型锚定" "世界建立" "冲突升级" "奇观爆发" "品牌锁定"
- 副标题：时间戳范围，格式 "MM:SS – MM:SS"

**右卡片（观众效果 — 心理目标）**
- 主标题：该段落试图在观众心中制造的核心效果（≤10字）
  例如："类型识别+注意锁定" "世界好奇心" "紧迫感+不安" "视觉承诺兑现" "记忆锚定+行动意图"
- 副标题：该效果对应的修辞诉求类型（≤14字）
  例如："genre rhetoric·类型识别" "story rhetoric·叙事好奇" "star rhetoric·明星认同"

> **填充原则**：
> 1. 左列写"预告片选择展示了什么"——锚定 MODULE 2（信息清单）+ MODULE 3（世界切片）
> 2. 中列写"这一段在修辞弧线中干什么"——锚定 MODULE 6（结构弧线）
> 3. 右列写"观众因此被制造了什么效果"——锚定 MODULE 6.3（预设读者）+ MODULE 2.2（谜面清单）
> 4. 如果某段落的核心操作是**遮蔽而非展示**（如刻意用快切使信息无法被读取），
>    左列卡片的主标题写"（信息遮蔽）"，副标题描述遮蔽手段
> 5. 如果某段落的观众效果主要是**悬念/谜面**而非情感冲击，
>    右列卡片的副标题标注 "hermeneutic code·谜面悬置"

### 段落切分指引

按以下优先级寻找切分点（优先级从高到低）：

1. **音乐断裂点（music drop）**：音乐突然消失、风格突变、或从silence进入音乐的时刻——这几乎总是预告片最重要的修辞节点
2. **剪辑速率突变点**：从慢节奏切到快速蒙太奇、或从快切突然静止
3. **信息密度跃迁点**：从氛围镜头切到叙事信息释放、或从叙事切到奇观展示
4. **字幕卡/标题卡插入**：任何非叙事性文字元素的出现
5. **标题出现 + 上映日期**：标志修辞叙事的结束和品牌锁定的开始

典型的预告片段落划分：

| 预告片类型 | 典型段落数 | 典型结构 |
|----------|-----------|---------|
| Teaser（30–60s）| 2–3 段 | 氛围建立 → 核心谜面 → 标题锁定 |
| Theatrical Trailer（2–2.5min）| 4–5 段 | 世界建立 → 角色引入 → 冲突升级 → 奇观爆发 → 品牌锁定 |
| Final Trailer（2–3min）| 5–6 段 | 世界建立 → 角色/关系铺陈 → 冲突加深 → 高潮蒙太奇 → 情感收束 → 品牌锁定 |

### viewBox 高度计算公式

```
段落数 = N
viewBox 高度 = 50 + N × 90 + 20
```

示例：5 段 → 50 + 5×90 + 20 = 520
示例：3 段 → 50 + 3×90 + 20 = 340
示例：6 段 → 50 + 6×90 + 20 = 610

### 输出禁忌

- 禁止在 SVG 内部写解释性文字段落
- 禁止使用 `<style>` 块定义颜色（使用 `c-{color}` class）
- 禁止硬编码十六进制颜色值
- 禁止在任何 `<text>` 元素上省略 class 属性
- 禁止让卡片文字超出 168px 宽度（主标题≤10字，副标题≤16字）
- 禁止将两个不同段落设为同一颜色
- 垂直箭头必须三列都画，不可只画中列
- 禁止将水平箭头画为双向——展示→功能→效果的因果方向严格为 左→中→右
- 禁止在右列"观众效果"卡片中写画面描述——右列只写观众被制造了什么心理效果
- 禁止将右列写成正片剧情推测——观众效果是"预告片对观众做了什么"，不是"正片会讲什么"
